## API Reference

### ScrollArea

The `ScrollArea` component provides a scrollable container with automatic shadow indicators that show when content overflows. The shadows appear/disappear automatically based on scroll position. The component includes a thin, styled scrollbar using native browser scrollbar styling.

### Components

#### ScrollArea.Root

The root container that handles scrolling behavior and automatically renders shadow overlays when content overflows.

| Prop                   | Type        | Default                            | Description                                        |
| ---------------------- | ----------- | ---------------------------------- | -------------------------------------------------- |
| `topShadowGradient`    | `string`    | `'from-background to-transparent'` | Custom Tailwind gradient classes for top shadow    |
| `bottomShadowGradient` | `string`    | `'from-background to-transparent'` | Custom Tailwind gradient classes for bottom shadow |
| `className`            | `string`    | -                                  | Additional CSS classes                             |
| `children`             | `ReactNode` | -                                  | Child components (must include ScrollArea.Content) |

#### ScrollArea.Content

The scrollable content container with built-in scrollbar styling.

| Prop        | Type        | Default | Description            |
| ----------- | ----------- | ------- | ---------------------- |
| `className` | `string`    | -       | Additional CSS classes |
| `children`  | `ReactNode` | -       | The scrollable content |

## Basic Usage

```tsx
import { ScrollArea } from '@/registry/joyco/blocks/scroll-area';

function App() {
  return (
    <ScrollArea.Root className="h-64">
      <ScrollArea.Content>
        {/* Your scrollable content */}
        <div>Content here</div>
      </ScrollArea.Content>
    </ScrollArea.Root>
  );
}
```

## Customization

### Custom Shadow Gradients

You can customize the shadow appearance using the `topShadowGradient` and `bottomShadowGradient` props:

```tsx
<ScrollArea.Root
  className="h-64"
  topShadowGradient="from-fd-card to-transparent"
  bottomShadowGradient="from-fd-card to-transparent"
>
  <ScrollArea.Content>
    <div>Content here</div>
  </ScrollArea.Content>
</ScrollArea.Root>
```

### Scrollbar Styling

The component includes built-in scrollbar styling with a thin scrollbar. The scrollbar uses:

- `scrollbar-width: thin` - Thin scrollbar width
- `scrollbar-color: hsl(0 0% 50%)` - Gray scrollbar color

These styles are applied automatically to the `ScrollArea.Content` component.

## How It Works

The ScrollArea component uses:

- **Automatic Shadow Detection**: Shadows automatically appear/disappear when content overflows
- **Scroll Tracking**: Uses ResizeObserver and MutationObserver to detect content changes
- **Native Scrollbar**: Uses browser-native scrollbar with custom styling via CSS properties
- **Shadow Positioning**: Shadows are positioned absolutely and don't interfere with scrollable content

## Notes

- Shadows are positioned absolutely and don't interfere with scrollable content
- The component uses `pointer-events-none` on shadows so they don't block interactions
- Shadows automatically update on scroll, resize, and content mutation events
- The default gradients use your theme's background color (`from-background`) for seamless integration
- Scrollbar styling uses Tailwind arbitrary values for `scrollbar-width` and `scrollbar-color`
- All styling is done with Tailwind classes (no styled-jsx)
