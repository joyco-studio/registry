## Hover + focus variant

We call this `hocus`, it combines the `hover` and `focus-visible` variants. Use this to style elements that should be styled when hovered or focused.

```css title="globals.css"
@custom-variant hocus (&:is(:hover, :focus-visible));
```

## Set line-height per font size

Configure the line-height for each font size.

```css title="globals.css"
@theme {
  --text-tiny: 0.625rem;
  --text-tiny--line-height: 1.5rem;
  --text-tiny--letter-spacing: 0.125rem;
  --text-tiny--font-weight: 500;
}
```

Reference -> [font-size docs](https://tailwindcss.com/docs/font-size#customizing-your-theme)

## Ancestor state styling

Use the `in-*` variant to style an element based on an ancestor's state or class. This is useful for things like drawer overlays or conditional layouts.

```html
<div class="in-[.notifs-drawer-open]:opacity-100">
```

This targets the element when it's inside an ancestor with the `.notifs-drawer-open` class.

Reference -> [in-* variant docs](https://tailwindcss.com/docs/hover-focus-and-other-states#implicit-groups)
